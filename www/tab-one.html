The basic element in the Kubernetes API is a Pod. A Pod groups one or more containers with filesystems and kernel namespaces. A Pod is the basic building block of all applications built in Kubernetes. We will begin our exploration of the API by creating a Pod. This first Pod will simply combine an instance of a node.js server, and a container that synchronizes the file system with a git repository. 
apiVersion: v1
kind: Pod
# information about the pod itself
metadata:
  name: nginx-pod
  labels:
    role: frontend
# specification for the pod
spec:
  containers:
  # specification for the first container in the pod
  - name: nginx-container
    image: nginx
    ports:
    - containerPort: 80
